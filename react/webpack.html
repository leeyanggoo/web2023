웹팩 WEBPACK

1. webpack 개념

webpack은 JavaScript 애플리케이션을 위한 모듈 번들러입니다.

개발자가 모듈 시스템을 사용하여 JavaScript 코드를 작성할 수 있다는 것입니다.
이는 코드의 재사용성을 높이고, 코드의 구조를 더 명확하게 만들며, 복잡한 프론트엔드 환경에서의 개발 작업을 보다 용이하게 만듭니다.

Entry: 웹팩이 내부 종속성 그래프를 만드는 시작점입니다. 기본값은 ./src/index.js이지만 웹팩 설정에서 직접 지정할 수 있습니다.

Output: 번들링된 코드가 저장될 위치를 지정합니다. 기본적으로 ./dist/main.js에 저장되지만 웹팩 설정에서 직접 지정할 수 있습니다.

Loaders: 웹팩은 기본적으로 JavaScript 및 JSON 파일만 이해하지만 로더를 사용하면 다른 유형의 파일들을 모듈로 변환하여 웹팩이 처리하게 할 수 있습니다.

Plugins: 로더가 특정 유형의 모듈을 변환하는 데 반해, 플러그인은 번들 최적화, 자산 관리 및 환경 변수 주입 등과 같은 광범위한 작업을 수행합니다.

Mode: 웹팩에 내장된 환경별 최적화를 위해 development, production 및 none 중 하나를 설정할 수 있습니다.

2-4. webpack.config.js 설치

npx webpack을 작성하면 파일이 번들링 됩니다.

const path = require('path');

module.exports = {
    mode: 'production', // production, development
    entry: {
        index: './src/index.js'
    },
    output: {
        path: path.resolve(__dirname, 'public'),
        filename: 'js/[name].js'
    }
}
